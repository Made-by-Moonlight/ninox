ScrollbackBuffer symbol

2026-07-01 · by scrollback-fix

ScrollbackBuffer is the ring buffer behind TerminalPane that holds rows evicted from the live grid. It exists because tmux capture and the PTY stream can disagree about where a frame begins; the buffer is the single source of truth for anything above the viewport.

The load-bearing invariant: frames must pass through [[frame-alignment]] before the grid diff runs. Diffing unaligned frames re-emits rows at the redraw boundary, which is exactly the duplicate-row bug recorded in [[errors/scrollback-dup]].

  • Owned by [[TerminalPane::draw_grid]]; written on every PTY read.
  • Capacity is fixed at spawn; eviction is oldest-first, no reallocation.
  • Cursor-addressed TUI redraws (vim, htop) never touch it — see [[cursor-addressing]].

Defined in crates/ninox-app/src/components/terminal.rs.
